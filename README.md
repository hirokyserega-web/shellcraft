# ShellCraft

ShellCraft — распределённая система автоматического крафта для **ComputerCraft: Tweaked** (Minecraft 1.19+/1.20+, Forge и Fabric).

Core-компьютер хранит рецепты, сканирует хранилище, строит дерево зависимостей и раздаёт задачи черепахам-воркерам параллельно. UI с вкладками рисуется на мониторе, предметы показываются по-русски, всё автообновляется с GitHub.

## Возможности

- **Автоустановка** одной командой `wget run ...`.
- **Автообновление** с GitHub без потери локальных данных (рецепты, конфиг, логи).
- **Автоопределение периферии** — сундуки, мониторы, модемы, машины подхватываются сами.
- **Core/Worker роли** + rednet-протокол (`lib/net.lua`).
- **Хранилище** предметов: несколько сундуков/бочек, pushItems/pullItems.
- **Обучение рецептов**: разложил предметы как в верстаке → система сохранила рецепт.
- **Крафт по сохранённым рецептам** через черепах-воркеров.
- **Несколько воркеров параллельно** с очередью задач, балансировкой и повтором при падении.
- **Рекурсивный крафт по дереву зависимостей** — промежуточные предметы крафтятся автоматически.
- **Калькулятор потребности** — показывает, чего и сколько не хватает по всему дереву.
- **Спец-механизмы** (печь, доменная печь, коптильня): положить вход → дождаться → забрать результат. Своя вкладка в UI.
- **UI с вкладками** на мониторе, тач-управление: Крафт, Хранилище, Механизмы (динамически), Рецепты, Лог.
- **Локализация**: ID → русские имена через `lang/ru.lua`, красивый fallback, лог отсутствующих переводов.
- Работает и в **CraftOS-PC** для отладки.

## Установка

На компьютере или черепахе выполните:

```lua
wget run https://raw.githubusercontent.com/hirokyserega-web/shellcraft/main/install.lua
```

Скрипт спросит роль (`core` или `worker`), скачает все файлы, создаст `config.dat` и настроит автозапуск. После установки перезагрузите компьютер/черепаху (`reboot`) — ShellCraft запустится автоматически.

> Если запускаете с форка — поменяйте `USER` в начале `install.lua`.

## Схема сборки в игре

### Core (главный компьютер)

Поставьте:

- **Computer / Advanced Computer**;
- **wired modem** (для rednet с воркерами и доступа к инвентарям по сети);
- **монитор** (для UI);
- **сундуки/бочки** рядом, подключённые к проводной сети (хранилище);
- при желании **печь / доменную печь / коптильню** — они попадут во вкладку «Механизмы».

Core сам найдёт хранилища, мониторы, модемы, машины и воркеров в сети.

### Worker (черепаха-крафтер)

Каждая воркер-черепаха должна иметь:

- **wired modem** — Core кладёт ингредиенты прямо в инвентарь черепахи и забирает результат через `pushItems`/`pullItems` по сети;
- принадлежность к той же проводной сети, что и Core;
- свободный крафт-инвентарь (слоты 1–16).

Черепахе **не нужны** свои сундуки-буферы — Core манипулирует её инвентарем напрямую по проводному модему.

## Как добавить воркеров

1. Поставьте черепаху, подключите её проводным модемом к сети Core.
2. Выполните на ней команду установки (см. выше), выберите роль `worker` и укажите ID компьютера Core.
3. Перезагрузите черепаху.
4. Она сама зарегистрируется в Core через rednet. Core начнёт раздавать ей задачи параллельно с другими воркерами.

ID Core-компьютера виден на его мониторе в строке статуса или в `config.dat`.

## Обучение рецептов

1. Перейдите во вкладку **Рецепты** на мониторе Core.
2. Нажмите **+ Обучить**.
3. На черепахе-обучателе разложите предметы в слотах 1–9 как в верстаке.
4. Нажмите **Готово** на мониторе.
5. Система выполнит крафт, считает ID и количество результата и сохранит рецепт (`shaped`).

Для рецептов через механизм (печь и т.п.) используйте тип `machine` — Core обработает его сам через вкладку «Механизмы».

## Как работает автообновление

При запуске `startup.lua` сначала вызывает `updater.lua`:

1. читает локальный `version`;
2. сравнивает с `version` на GitHub;
3. если новее — скачивает все `.lua` файлы;
4. **не трогает** защищённые файлы (`config.dat`, `recipes.dat`, `config.local.lua`, `lang/missing.txt`, `shellcraft.log`);
5. перезагружает компьютер (`os.reboot`).

## Заказ крафта и дерево зависимостей

1. Вкладка **Крафт** → тап по рецепту → выбор количества.
2. Калькулятор показывает потребность по всему дереву: что есть (зелёным) и чего не хватает (красным).
3. **ОК** — Core строит дерево зависимостей и ставит шаги в очередь (от листьев к корню).
4. Промежуточные предметы крафтятся автоматически, финальный — последним.
5. Если базового ресурса не хватает — заказ отклоняется с точным указанием, чего и сколько не хватает.

## Локализация

ID приходят на английском (`minecraft:oak_planks`). Система показывает их по-русски через `lang/ru.lua`. Если перевода нет:

1. берётся `displayName` из `getItemDetail`;
2. иначе ID форматируется красиво (`Oak planks`);
3. отсутствующий ID дописывается в `lang/missing.txt` — удобно дозаполнить словарь.

## Структура репозитория

```
shellcraft/
├── install.lua          — установка одной командой
├── startup.lua          — апдейт + запуск роли (core/worker)
├── updater.lua          — автообновление с GitHub
├── config.lua           — автогенерируемый конфиг + автоопределение периферии
├── version              — версия проекта
├── core/
│   ├── server.lua       — главный сервер (parallel-корутины)
│   ├── storage.lua      — хранилище, pushItems/pullItems
│   ├── recipes.lua      — рецепты + обучение
│   ├── planner.lua      — дерево зависимостей, калькулятор
│   ├── machines.lua     — спец-механизмы (печь и т.п.)
│   └── dispatcher.lua   — раздача задач воркерам
├── worker/
│   └── worker.lua       — черепаха-крафтер
├── ui/
│   ├── ui.lua           — вкладки, отрисовка, тач
│   └── widgets.lua      — рамки, кнопки, списки
├── lang/
│   └── ru.lua           — словарь переводов + localize()
└── lib/
    ├── net.lua          — rednet-протокол
    └── util.lua         — логи, файлы, форматирование
```

## Технические детали

- Только API CC: Tweaked: `turtle`, `peripheral`, `fs`, `os`, `rednet`, `term`, `parallel`, `textutils`, `colors`, `http`.
- Перемещение предметов — `pushItems`/`pullItems` через проводные модемы.
- UI и сетевой слушатель работают параллельно через `parallel.waitForAny`.
- Ошибки (нет ингредиента, воркер недоступен, машина занята) логируются и показываются в UI.
